# Typescript email ui
## Function
Create the web UI under `src/ui/` (React + Vite + TypeScript) and the Capacitor shell config (`capacitor.config.ts`); do not execute! The UI is the client of the Proxy pattern three times over: every screen obtains a `MailProvider` from the `ProviderRegistry`, a `MailIntelligence` (see `typescript_mail_intelligence.md`), and a `MailStore` (see `typescript_mail_store.md`) from the composition root, and calls only interface methods â€” it must compile and pass all tests with nothing but the in-memory `FakeProvider`, `FakeIntelligence`, and `FakeMailStore`, proving that adding IMAP, a different AI backend, or a different storage engine later requires zero UI changes. Synced mail persists in the store (SQLite on device), so reading and text search work offline. **AI is optional**: when configured, arriving threads are auto-classified and tagged, long threads open with a digest, compose starts from a draft, and search interprets natural language; without AI configured, the app remains fully functional for reading, tagging, and sending mail. The Capacitor shell wraps the built web app for iOS/Android with native OAuth for Gmail sign-in; native platform setup commands (`npx cap add ios|android`) are documented in the README, not executed.
